# StudyT2C 서비스 목적 정리 & MVP 정리·고도화 제안

> **포지셔닝**: 기존 온라인 학습 솔루션은 기능이 많지만 **오프라인 연계가 약함**. StudyT2C는 **기존 오프라인 수업을 지지하는 보조 솔루션**이며, **학원 관리 프로그램이 아님**.  
> **본질**: 숙제·복습 과정에서 학생별 **강점·취약점을 분석** → **오프라인 수업의 개인화·맞춤 수업** 가능 → **학부모 신뢰 상승**, **학원은 데이터 기반으로 학생 성취도 극대화**.

---

## 1. 서비스 목적 (한 문장)

**오프라인 수업을 유지한 채, 숙제·복습 데이터로 학생별 개인화를 높이고, 학부모에게 학습 성향·평가·시간에 따른 성취도를 보여 주며, 학원이 그 데이터로 수업 성취도를 극대화하도록 돕는 보조 솔루션.**

---

## 2. 인터넷 조사 요약 — “오프라인 보조·개인화” 관점에서 참고할 점

### 2.1 국내·해외 유사 포지셔닝

| 출처 | 핵심 메시지 |
|------|-------------|
| **클래스팅** | AI 진단 + 수준별 학습, **교사 중심 대시보드**로 학급 전체·개별 취약 단원 실시간 확인. |
| **콴다수학코치** | 학원 대상. **숙제·테스트 결과 → 학생별 학습 리포트·맞춤 문제** 자동 생성, 채점·숙제 검사·출결 등 **운영 업무 자동화** (관리보다는 학습 데이터에 초점). |
| **캐츠(CATS)** | **학습 맞춤형 리포트**로 학원 경쟁력·학부모 만족도 상승. 단원별 전국 평균·상위 커트라인으로 **상대적 위치** 제공, 선생님이 **성적 리포트로 학생별 수준 맞춤 문제** 제공. |
| **Kiddom Atlas** | 학생 작업을 **매일 분석**해 오개념·격차 식별 → **다음 날 수업용 타깃 자료** 자동 생성. “어디서 막히는지” + **다음 수업 조정 권고**. |
| **National Student Support Accelerator** | 튜터링은 **Mastery Data**(평가·숙제·블렌디드 소프트웨어 성과) 수집 → **다음 세션 개인화**. “한 세션의 exit ticket → 다음 세션 맞춤” 사이클. |
| **Formative / Edmentum** | **형성평가 데이터**로 즉시 수업 방향 전환·차별화. 효과 크기 0.4–0.7. “다음 수업 준비”를 **증거 기반**으로. |
| **Acadience / LAC Parent Access** | **학부모용 리포트**는 “무엇을 측정했는지·기술이 무엇인지” 명확히, 시각화·언어 선택으로 **신뢰·투명성** 강화. |

### 2.2 정리된 원칙

- **오프라인 보조**: 수업 자체를 대체하지 않고, **숙제·채점·복습 데이터**가 **다음 오프라인 수업**에 쓰이게.
- **선생님 액션**: “취약점 나열”만이 아니라 **다음 수업에서 무엇을 보강할지** 권고·요약이 있으면 좋음.
- **학부모 신뢰**: **학습 성향·평가·시간에 따른 성취도**가 한눈에 보이고, **리포트가 상대적 위치**(단원별·전국 비교 등)를 담으면 만족도·신뢰 상승 사례 있음.
- **MVP는 단일 시나리오**: “학원 관리”보다 **한 학생의 숙제·채점·오답 → 리포트 → 선생/학부모** 흐름에 집중하는 것이 검증에 유리.

## 3. 현재 MVP에서 “본질에 비해 상대적으로 덜 중요한” 기능·UI (간소화 권장)

아래는 **“오프라인 수업 보조·학생별 개인화·학부모 신뢰”**에 직결되지 않는 부분으로, **간편·깔끔하게 정리**해도 되는 후보입니다.

| 구분 | 현재 기능/UI | 권장 방향 | 이유 |
|------|--------------|-----------|------|
| **학생** | 학습/숙제/오답노트/기록 4탭 동등 노출 | **학습(대시+튜터+채점)**을 메인으로, 숙제·오답노트·기록은 서브 또는 접기 | 오프라인 보조의 핵심은 “숙제·채점·복습 데이터 생성”; 이게 쌓이는 화면이 메인이어야 함. |
| **학생** | 수업 중/쉬는 시간 모드, 공부 외 질문 제한 | 유지하되 **설정은 한 곳·최소 노출** | 집중/품질 관리에 도움되지만, UI가 복잡해지면 본질(개인화 데이터)이 가려짐. |
| **학생** | 개발 모드·이미지 크게 보기 등 | **사이드바 접이식**에만 두거나 제거 | 본질과 무관. |
| **학부모** | 공부 외 질문 모니터링·탭 이탈 알림 | **알림 자체는 유지**, UI는 “요약 뱃지 + 상세는 접기” | 학부모 신뢰·집중도 가시화에 기여하지만, 화면을 과하게 차지하지 않게. |
| **학부모** | 이메일·주간·일일·실시간 등 알림 옵션 다수 | **기본 1~2가지**(예: 주간 리포트 + 탭 이탈 실시간)로 단순화 | 옵션이 많으면 “설정 부담”만 커질 수 있음. |
| **선생님** | 반 대시보드 vs 학생별 상세 | **학생별 상세(개인화 데이터)**를 1차, 반 보드는 요약만 | “학원 관리”가 아니라 “학생별 개인화”가 본질이므로, 학생 한 명씩 보는 뷰가 메인. |
| **선생님** | 상담·숙제·요약 탭 | **AI 분석(취약점·성취도)**를 가장 앞에, 상담/숙제/요약은 그다음 | 다음 수업 준비에 바로 쓰는 건 “이 학생의 취약점·성취도”이기 때문. |
| **공통** | 개발자용 JSON/디버그 노출 | **개발 모드 시에만** 또는 제거 | 일반 사용자에게는 본질 아님. |

## 4. 본질에 맞게 “고도화”하면 좋은 기능 (조사 기반 제안)

아래는 **오프라인 수업 보조·개인화·학부모 신뢰**를 위해 **우선 고도화**할 만한 항목입니다.

### 4.1 선생님·학원 — “다음 수업 준비” 액션 아이템

| 고도화 방향 | 설명 | 참고(조사) |
|-------------|------|------------|
| **학생별 “다음 수업 권고” 한 줄** | 취약 단원·오답 유형을 바탕으로 “이번 주에는 ○○ 단원 보강 권장” 같은 **액션 가능한 문장** 제공. | Kiddom Atlas(다음 날 수업용 자료), NSSA(Mastery Data → 다음 세션 개인화), Formative(형성평가 → 수업 전환). |
| **단원/개념 단위 취약점** | 현재 “오답 유형”(개념/계산/독해 등)을 **단원·개념(키 컨셉)**까지 쪼개서 보여 주기. | 클래스팅(취약 단원), 캐츠(단원별 리포트), 오답 분석 3단계(유형 파악 → 해결 방법 정리). |
| **반 전체 vs 개인** | 반 전체 공통 취약 단원 + **학생별 상위 3개 취약점** 요약. “이 학생만 따로 보강” 판단에 사용. | Formative(differentiate next steps), 클래스팅(학급 전체 + 개별). |

### 4.2 학부모 — 신뢰·가시화

| 고도화 방향 | 설명 | 참고(조사) |
|-------------|------|------------|
| **상대적 위치(선택)** | 단원별·유형별 “전국/반 평균 대비” 또는 백분위 간단 표시. | 캐츠(전국 평균·상위 커트라인), Acadience(학부모용 명확한 측정 지표). |
| **한 장 요약 이미지** | 주간/월간을 **한 장 그림·PDF**로 내보내기. 공유·저장 용이. | AI생활기록부·밀크T 등 “한 장 요약” 사례. |
| **학습 성향·추이 문구** | “최근 N주간 질문 수·오답률 추이·자주 틀리는 유형”을 **문장으로 요약**(예: “계산 실수 비율이 높아졌어요”). | 학부모는 숫자보다 **이해하기 쉬운 문장** 선호 사례. |

### 4.3 학생 — 개인화 데이터 “쌓이게” 하기

| 고도화 방향 | 설명 | 참고(조사) |
|-------------|------|------------|
| **오답 → “다음 복습 시점”** | 오답노트에 **간격 반복(리뷰 일정)** 노출. “이 문제는 N일 후 다시” 같은 짧은 안내. | FSRS/Anki, 오답 3일 이내 복습 효과. |
| **취약 개념 한 줄** | 채점·오답 저장 시 **key_concepts**를 학생에게도 “이번에 부족했던 개념: ○○”처럼 한 줄로 노출. | 맞춤형 복습·메타인지. |
| **숙제·채점 진입점 단순화** | “오늘 할 일: 숙제 1건, 복습 3문항”처럼 **한 화면에서 다음 액션** 제안. | 본질은 “숙제·복습 데이터 생성”; 진입이 쉬워야 데이터가 쌓임. |

### 4.4 데이터·리포트 일관성

| 고도화 방향 | 설명 | 참고(조사) |
|-------------|------|------------|
| **오개념 vs 실수 구분** | 오답 원인을 “개념 부족(오개념)” vs “계산 실수/독해 실수” 등으로 구분해, **선생님 개입 방식**이 달라지게. | Misconception vs mistake 구분(수학 피드백 연구), 오답 유형별 대응. |
| **기간·단위 통일** | 주간/월간, “최근 30일” 등 **기간 정의를 화면마다 동일**하게 하고, “이번 주/이번 달” 라벨을 명확히. | 학부모·선생님 혼동 방지, 신뢰도 상승. |

## 5. 우선순위 제안 (요약)

- **먼저 정리(간소화)**:  
  - 학생 탭 구조(학습 메인, 나머지 서브/접기), 학부모/선생 알림 옵션 단순화, 개발/디버그 노출 최소화, 선생님 화면에서 “학생별 개인화”를 1차로.
- **먼저 고도화**:  
  - **선생님**: 학생별 “다음 수업 권고” 한 줄 + 단원/개념 단위 취약점.  
  - **학부모**: 학습 성향·추이 문장 요약 + (가능하면) 한 장 요약.  
  - **학생**: 숙제·채점·복습 진입 단순화 + 취약 개념 한 줄·복습 시점 안내.

이 순서로 적용하면 **“기존 오프라인 수업을 지지하며, 학생별 개인화와 학부모 신뢰·학원 성취도 극대화”**라는 서비스 목적에 맞게 MVP가 정리·고도화됩니다.
